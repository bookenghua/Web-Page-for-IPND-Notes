Lesson I: The Basics

How the Web Works

The web is not dissimilar to a marketplace where various agents congregate to exchange goods, services and ideas. There are, however, two main differences.

1. Distance is a non-factor.

2. Computers act as proxies for their human counterparts.

In other words, computers are taught to communicate with one another in a way which makes content not only intelligible to another computer but also meaningful to humans. Accessing a web page from a computing device seems deceptively simple. In reality, a multitude of processes are at work. When a person enters "www.google.com" into their web browser (on a computer/mobile computing device), their computer sends a HTTP Request - which is really one of many file transfer protocols - to a server. The server then finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

It is useful to conceptualize the web as a cloud - a concentration of uncountable nodes - where the nodes are linked by the file transfer protocols. These links are probably where the synechdoche of internet as information superhighway originated from.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain text content independent of markup. The former can be thought of as plain text while the latter modifies the plain text and changes how it looks.

Elements, Tags and Attributes

HTML documents are made of HTML elements. HTML elements are in turn comprised of tags and their respective contents. When writing HTML, we tell browsers the type of each element by using HTML tags such as

Inline Elements 
            <br> - break
            <em> content </em> - emphasis/italics
            <img> - image
            <a> content </a> - hyperlink
            <span> content </span> - grouping
Block Elements

            <p> content </p> - paragraph
            <div> content </div> - division

Inline elements leave the structure and whitespace surrounding plain text alone while block elements form an "invisible box" around the content inside of them. It should also be noted that <br> is a void tag as there is no corresponding closing tag.

Within particluar tags, there may exist attributes which serve to further modify the instructions given by the tags which eventually modify the content. Exmaples include 

            src
            height
            href
Key Takeaways

Computers need to be told exactly what to do with extremely specific sets of instructions such that the communication process is made intelligible to all parties involved.

Lesson II: HTML and CSS as languages

A Simple Analogy

HTML, CSS and Javascript are all languages used to perform different functions. Thinking about a house is useful. HTML can be thought of as the structure and general backbone of the house, CSS as the color of walls and rugs and Javascript as the garage door opener. Relating these ideas back to a webpage, HTML provides the structure of the webpage, CSS makes things interesting by providing exciting fonts and splashes of strategically placed colors while Javascript does the heavy lifting in terms of interactions with the user.

Covered in the next section will be yet another analogy - a tree diagram to represent the Document Object Model (DOM)

The Document Object Model (DOM)

Frequently referred to as DOM and more endearingly termed as the 'tree-like structure' by developers, this is a very important concept that aids our conceptual understanding about boxifying HTML elements. Similar to how a Russian Matryoshka doll has an almost infinite (well, not exactly but you get the idea) number of dolls within the mother doll, the indentation of HTML elements based on their opening and closing tags is interpreted by the browser with the help of DOM. The more indented the element, the more deeply it is nested.

The pencil is your best friend

When writing code, always fall back on a trusty pencil when the going gets tough. Although it seems quite easy to boxify stuff in your mind, it gets pretty daunting after a while and it's easy to lose track of which tag belongs to which element. Things don't need to be that complex. Planning in advance with a pencil always helps.

Lesson III: CSS - versatile and powerful

Avoiding repetition is key

CSS stands for Cascading Style Sheets. It allows web programmers to add styles to a HTML document in a layered manner, where the most specific rule is applied. It therefore follows that HTML elements should be expressed in the most elegant manner such that style paramenters are defined within nested classes so that a simple change in the CSS file affects all targeted classes. In other words, avoid repetition.

Understanding documentation

Thinking like a programmer does not mean one has to memorize all the different types of CSS attributes and possible values that are applicable. What is more important is to grasp the general idea of each concept and then source for the necessary documentation whenever an unknown scenario crops up. Programmers must possess not only a good conceptual understanding of HTML and CSS but also be able to search for and interpret key pieces of documentation when in doubt.

For instance, it sure is necessary for programmers to understand and recall basic stuff like the class having no periods or punctuation marks. Similarly, one would do well remembering the box model where, from the outside in, an element consists of its margin, border, padding and content. Specific attributes like font-style and font-weight and their values can be obtained from a quick reference to existing documentation.
